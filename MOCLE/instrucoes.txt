Para compilar e executar o mocle, considerar as seguintes informações:

- precisa da boost

- precisa da biblioteca de particionamento de grafos metis, que pode ser obtida 
em http://glaros.dtc.umn.edu/gkhome/views/metis

na versão anterior do mocle eu tinha mandado o executavel dessa biblioteca 
que era necessário, mas que agora não funciona mais. O que vai precisar 
dessa biblioteca é o gpmetis. O que fiz para usar no linux foi compilar a 
biblioteca conforme as instruções do proprio site e depois copiar o arquivo 
executavel do gpmetis junto com o codigo do mocle. Para o windows acredito 
que voces terão que fazer algo semelhante, ou ajustar os paths para executar.

- no inicio do main, trocar o comentário das linhas:
//    runningSystem = mywindows;
    runningSystem = mylinux;

- depois de compilado, para a execução a linha de comando é a seguinte:

mocle <Crossover Type: 1- mcla or 2 - bipartite> <Dataset> <Initial Population Directory> <Results Directory> <True Partition (can be a fake one if not known)>

no último parametro deve ser indicada uma partição conhecida dos dados, mas esse parametro não é usado na execução usual do mocle. Mas como o codigo é antigo, teria muitas mudanças a serem feitas para remover esse parametro, e não tive tempo de alterar. Assim, para executar voces podem simplesmente usar uma das partições produzidas para a população inicial, por exemplo.
